# Resource Model

A Resource represents anything displayed on the dashboard.

## Core fields

- ID
- Name
- Description
- Group
- Tags
- Health
- Actions

## Actions

Actions represent entry points associated with the resource.

Examples:

- Open
- Admin
- Documentation
- Repository

## Rendering

A Resource may be rendered using different presentation modes.

- Compact
- Standard
- Detailed

Rendering never changes the Resource itself.
